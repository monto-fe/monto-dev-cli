// 如何使用

- 1、安装 CLI（全局、本地）
- 2、yarn um-dev-cli mock (直接本地新建 mock 目录)
- 3、支持 Action
- 4、支持 json 文件配置 mock 规则
- 5、支持 mock 数据存放在其他地方
- 6、根据接口直接生成代码，
  - 可以直接生成组件
  - 也可以生成对应的项目自行启动
  - 路径默认是 mock 平级路径下新生成 monto-component
- 7、生成的模版，需要给模版添加配置文件，在拉取的时候，判断下用户当前的环境是否和模版匹配，如果匹配在进行拉取
